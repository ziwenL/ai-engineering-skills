# Skill Writing Guide

一个标准 Skill 应包含：

```text
frontmatter:
  name
  description

正文：
  适用场景
  输入
  工作流程
  输出格式
  约束
  不适用场景
```

原则：

1. 描述要具体。
2. 流程要可执行。
3. 输出要结构化。
4. 约束要明确。
5. 不写入临时信息和敏感信息。
6. 如果 Skill 目录下有 `checklist.md`、`architecture.md`、`log-analysis.md` 等辅助文件，必须在 `SKILL.md` 中明确要求读取和使用。
7. 不要只写“检查项列表”，要说明先读什么、怎么做、最后怎么输出。
8. 开发类 Skill 必须写清验证闭环：至少说明需要运行的编译、测试、Lint 或人工验证方式。
9. 审查类 Skill 必须写清证据要求、风险分级和是否阻塞合并或发版。

推荐章节模板：

```text
## 适用场景
## 输入
## 工作流程
## 输出格式
## 约束
## 不适用场景
```
