# 技术方案 Skill

## 适用场景

- 需求分析确认后的技术方案设计
- 复杂改动前的模块影响和风险评估
- 需要先明确文件改动、接口变化和验证方式

## 输入

- 已确认的需求分析
- 项目中相似实现
- 当前架构和模块边界

## 工作流程

1. 查找项目中相似实现。
2. 判断涉及模块。
3. 设计状态流转。
4. 设计错误处理。
5. 评估接口和数据结构变化。
6. 设计测试方案。
7. 识别风险和兼容性问题。
8. 给出最小可行实现路径。

## 输出格式

```text
【总体方案】
【相似实现】
【涉及模块】
【修改文件清单】
【数据结构】
【接口变化】
【状态流转】
【错误处理】
【测试方案】
【风险与兼容】
【是否需要灰度/开关】
```

## 约束

- 不要直接进入代码实现。
- 明确哪些是事实，哪些是假设。
- 优先最小可行方案，避免无关重构。

## 不适用场景

- 需求还没有分析清楚
- 只是很小的纯文案改动，不需要独立技术方案
