# 事故应急处理 Skill

## 适用场景

- 线上事故进行中
- 需要止血、回滚、降级或限流
- 需要快速明确通知对象和后续动作

## 输入

- 当前事故现象
- 影响范围
- 最新发版和配置变更
- 当前可执行的开关、回滚、降级手段

## 工作流程

1. 判断事故等级。
2. 明确影响范围和用户影响。
3. 判断是否需要回滚、关闭开关、降级、限流。
4. 明确通知对象。
5. 先止血，再修复。
6. 记录时间线。
7. 输出下一步责任人和动作。

## 输出格式

```text
【事故等级】
【影响范围】
【当前状态】
【止血方案】
【回滚/降级建议】
【通知对象】
【下一步动作】
【需要记录的时间线】
```

## 约束

- 先止血，再优化根因分析。
- 回滚、降级、限流、关开关等高风险操作必须人工确认。
- 明确区分当前已执行动作和建议动作。

## 不适用场景

- 事故已经结束、只需要复盘的阶段
- 没有线上影响的普通需求开发
