# 使用示例

## 说明

这个 Skill 适合在事故仍在进行时，先明确影响范围、止血方案、回滚或降级建议、通知对象和下一步动作。

## 最小调用示例

```text
请使用 incident-response skill，基于当前线上问题给出应急处理方案。

请输出：
1. 事故等级
2. 影响范围
3. 当前状态
4. 止血方案
5. 回滚 / 降级建议
6. 通知对象
7. 下一步动作
8. 需要记录的时间线

要求：
- 先止血，再优化根因分析。
- 高风险操作必须标记为需要人工确认。
```

## 推荐调用示例

```text
请使用 incident-response skill，基于当前线上事故给出应急处理方案。

请基于以下输入：
1. 当前事故现象
2. 影响范围
3. 最近发布和配置变更
4. 当前可执行的开关、回滚、降级或限流手段

请输出：
1. 事故等级
2. 影响范围
3. 当前状态
4. 止血方案
5. 回滚 / 降级建议
6. 通知对象
7. 下一步动作
8. 需要记录的时间线

要求：
- 先止血，再修复。
- 区分“已执行动作”和“建议动作”。
- 回滚、降级、限流、关开关等高风险操作必须人工确认。
```

## 高约束调用示例

```text
请使用 incident-response skill，严格给出当前事故的应急方案。

请严格按以下结构输出：
【事故等级】
【影响范围】
【当前状态】
【止血方案】
【回滚 / 降级建议】
【通知对象】
【下一步动作】
【需要记录的时间线】

要求：
- 先止血，再做深入根因分析。
- 必须区分已执行动作和建议动作。
- 所有高风险操作必须标记为需要人工确认。
```

## 适用提醒

- 适合事故仍在进行中时使用。
- 如果事故已经结束并进入复盘阶段，应切到 `postmortem`。
- 这个 Skill 不是用来直接做长期根因沉淀的。
