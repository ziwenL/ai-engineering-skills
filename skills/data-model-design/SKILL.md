# 数据模型设计 Skill

## 适用场景

- 数据库表设计或变更
- DTO / VO / Entity / 客户端 Model 设计
- 缓存结构和迁移方案设计

## 输入

- 业务实体和字段需求
- 读写场景
- 兼容性和迁移约束

## 工作流程

1. 明确业务实体。
2. 设计字段和类型。
3. 设计字段兼容性和默认值。
4. 判断是否需要数据库迁移。
5. 设计 DTO / VO / Entity 映射。
6. 设计缓存结构。
7. 输出迁移和回滚方案。

## 输出格式

```text
【业务实体】
【字段与类型】
【约束与默认值】
【映射关系】
【缓存设计】
【迁移方案】
【回滚方案】
【风险点】
```

## 约束

- 数据库迁移必须先出方案。
- 不允许直接修改生产数据。
- 字段删除必须给兼容方案。

## 不适用场景

- 纯 UI 调整
- 与数据结构无关的轻量文案修改
