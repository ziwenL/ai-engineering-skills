# 数据模型设计 Skill

## 适用场景

- 数据库表设计或变更
- DTO / VO / Entity / 客户端 Model 设计
- 缓存结构和迁移方案设计

## 输入

- 业务实体和字段需求
- 读写场景
- 兼容性和迁移约束
- `usage-examples.md`

## 工作流程

1. 明确业务实体、字段职责和读写路径。
2. 设计字段和类型，补齐默认值、可空性、索引和唯一性约束。
3. 设计 DTO / VO / Entity / 客户端 Model 的映射关系。
4. 判断是否需要数据库迁移、缓存调整和回滚方案。
5. 明确兼容性影响，例如旧字段保留、灰度迁移、双写 / 双读策略。
6. 给出模型设计的验证方式，例如迁移演练、Schema 校验、兼容性验证。
7. 标记尚未确认的字段语义、迁移假设或未验证项。

## 输出格式

```text
【业务实体】
【字段与类型】
【约束与默认值】
【映射关系】
【缓存设计】
【迁移方案】
【回滚方案】
【验证方式】
【未验证项】
【风险点】
```

## 约束

- 数据库迁移必须先给方案。
- 不允许直接修改生产数据。
- 字段删除必须给兼容方案。
- 没有验证路径时，不要把迁移写成低风险。

## 不适用场景

- 纯 UI 调整。
- 与数据结构无关的轻量文案修改。

## 使用示例

```text

请输出：

- 数据库迁移必须先给方案。
- 字段删除必须给兼容方案。
```
