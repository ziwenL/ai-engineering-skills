# 使用示例

## 说明

这个 Skill 适合在涉及数据库表、DTO、VO、Entity、客户端 Model、缓存结构或迁移方案时，先把数据模型和兼容迁移路径设计清楚。

## 最小调用示例

```text
请使用 data-model-design skill 设计当前需求的数据模型。

请输出：
1. 业务实体
2. 字段与类型
3. 约束与默认值
4. 映射关系
5. 缓存设计
6. 迁移方案
7. 回滚方案
8. 验证方式
9. 未验证项

要求：
- 数据库迁移必须先给方案。
- 字段删除必须给兼容方案。
```

## 推荐调用示例

```text
请使用 data-model-design skill，为当前需求设计数据模型和迁移方案。

请先明确：
1. 业务实体和字段需求
2. 读写场景
3. 兼容性和迁移约束

请输出：
1. 业务实体
2. 字段与类型
3. 约束与默认值
4. 映射关系
5. 缓存设计
6. 迁移方案
7. 回滚方案
8. 验证方式
9. 未验证项
10. 风险点

要求：
- 明确字段可空性、索引、唯一性和默认值。
- 如果涉及 DTO / VO / Entity / 客户端 Model，请说明映射关系。
- 没有验证路径时，不要把迁移写成低风险。
```

## 高约束调用示例

```text
请使用 data-model-design skill，严格设计当前需求的数据模型。

请严格按以下结构输出：
【业务实体】
【字段与类型】
【约束与默认值】
【映射关系】
【缓存设计】
【迁移方案】
【回滚方案】
【验证方式】
【未验证项】
【风险点】

要求：
- 数据库迁移必须先给方案。
- 不允许直接修改生产数据。
- 字段删除必须配套兼容与回滚思路。
```

## 适用提醒

- 适合数据结构要变动时使用。
- 如果只是 UI 调整或纯前端交互改动，不适合用它。
- 如果接口结构也要一起设计，通常会和 `api-design` 配合使用。
