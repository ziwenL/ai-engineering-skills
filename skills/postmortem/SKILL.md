# 事故复盘 Skill

## 适用场景

- 线上事故结束后的复盘
- 需要沉淀时间线、根因、改进项和长期防线

## 输入

- 事故时间线
- 影响范围
- 止血和修复过程
- 已确认事实和证据

## 工作流程

1. 梳理事故时间线和影响范围。
2. 区分已确认事实、直接原因和系统性原因。
3. 总结处理过程中的有效动作和不足。
4. 提炼短期改进和长期防线。
5. 判断哪些内容需要沉淀到 Skills 或项目文档。

## 输出格式

```text
【事故摘要】
【影响范围】
【时间线】
【根因分析】
【处理过程】
【做得好的地方】
【问题与不足】
【短期改进】
【长期防线】
【需要沉淀到 Skills 的内容】
```

## 约束

- 不甩锅，聚焦系统改进。
- 没有证据时不要写成确定根因。
- 改进项要尽量可执行，而不是停留在口号。

## 不适用场景

- 事故仍在进行中
- 只有模糊传闻、没有基本时间线和事实的场景
