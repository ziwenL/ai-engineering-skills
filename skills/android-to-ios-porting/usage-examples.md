# 使用示例

## 说明

这个 Skill 适合在 Android 已有功能实现、iOS 已有可承接结构的前提下，把平台无关逻辑提炼出来并复刻到 iOS。

## 最小调用示例

```text
请使用 android-to-ios-porting skill，根据 Android 已实现功能复刻 iOS。

请输出：
1. Android 现有行为
2. 平台无关逻辑
3. iOS 涉及模块
4. 需要修改的文件
5. 行为一致性要求
6. 验证方式
7. 未验证项

要求：
- 不要机械翻译 Android 代码。
- 行为一致优先于结构一致。
```

## 推荐调用示例

```text
请使用 android-to-ios-porting skill，把当前 Android 已有功能复刻到 iOS。

请先阅读：
1. Android 已实现代码
2. iOS 当前架构和相似模块
3. 已确认的产品需求与交互要求
4. 已知平台差异和限制

请输出：
1. Android 现有行为
2. 平台无关逻辑
3. iOS 涉及模块
4. 需要修改的文件
5. 行为一致性要求
6. 允许保留的平台差异
7. 验证方式
8. 验证结果
9. 未验证项
10. 风险点

要求：
- 不要机械翻译 Android 代码。
- 优先复刻用户可感知行为。
- 如果某些一致性要求目前无法验证，必须明确写入未验证项。
```

## 高约束调用示例

```text
请使用 android-to-ios-porting skill，严格按多端一致性目标复刻当前功能。

必须先阅读：
1. Android 已实现代码
2. iOS 当前架构和相似模块
3. 已确认产品需求
4. 已知平台差异

请严格按以下结构输出：
【Android 现有行为】
【平台无关逻辑】
【iOS 涉及模块】
【需要修改的文件】
【行为一致性要求】
【允许保留的平台差异】
【验证方式】
【验证结果】
【未验证项】
【风险点】

要求：
- 不要机械翻译 Android 代码。
- 行为一致优先于结构一致。
- 没有证据时不要猜测 Android 的真实行为。
```

## 适用提醒

- 适合 iOS 已经有可承接结构的场景。
- 如果 iOS 工程是否可承接还不明确，应先用 `android-to-ios-bootstrap`。
- 如果需求本身仍未确认，不适合直接进入复刻实现。
