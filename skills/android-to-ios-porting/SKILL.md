# Android 到 iOS 复刻 Skill

## 适用场景

- Android 已有功能需要在 iOS 端复刻
- 多端行为需要对齐
- 需要从 Android 实现中提炼平台无关逻辑

## 输入

- Android 已实现代码
- iOS 端现有架构和相似模块
- 已确认的产品需求和交互要求
- 已知平台差异和限制
- `usage-examples.md`

## 工作流程

1. 阅读 Android 已实现代码，明确用户可感知行为。
2. 提取平台无关逻辑，包括用户流程、状态机、接口请求、响应处理、错误处理和埋点。
3. 查找 iOS 相似模块和可复用实现。
4. 列出必须保持一致的行为，以及允许保留的平台差异。
5. 按 iOS 架构实现，避免机械翻译 Android 代码。
6. 设计验证方式，覆盖主流程、异常流程和多端一致性比对。
7. 输出验证结果、未验证项和风险点。

## 输出格式

```text
【Android 现有行为】
【平台无关逻辑】
【iOS 涉及模块】
【需要修改的文件】
【行为一致性要求】
【允许保留的平台差异】
【验证方式】
【验证结果】
【未验证项】
【风险点】
```

## 约束

- 不机械翻译 Android 代码。
- 行为一致优先于结构一致。
- 证据不足时不要猜测 Android 真实行为。
- 无法验证的一致性要求必须明确写入“未验证项”。

## 不适用场景

- Android 端还没有稳定实现。
- 单端独有能力开发。
- 需求本身尚未确认。

## 使用示例

最小调用示例：

```text

请输出：
1. Android 现有行为
2. 平台无关逻辑
3. iOS 涉及模块
4. 需要修改的文件
5. 行为一致性要求
6. 验证方式
7. 未验证项

要求：
- 不要机械翻译 Android 代码。
- 行为一致优先于结构一致。
```

更多示例见 `usage-examples.md`。
