# 接口设计 Skill

## 适用场景

- 新接口设计
- 旧接口扩展或兼容改造
- 需要明确请求、响应、错误码和版本策略

## 输入

- 业务目标和调用场景
- 调用方类型
- 已有相似接口和兼容约束
- `usage-examples.md`

## 工作流程

1. 明确业务场景、调用方和接口目标。
2. 设计接口路径和方法。
3. 设计请求参数，包括必填、选填、默认值和校验规则。
4. 设计响应结构和错误码，覆盖主流程与异常流程。
5. 检查鉴权、权限、幂等、限流和审计要求。
6. 检查客户端兼容性、版本策略和弃用路径。
7. 给出示例请求 / 响应，以及接口设计的验证方式。
8. 标记需要和产品、客户端或服务端进一步确认的未验证项。

## 输出格式

```text
【接口目标】
【请求方法与路径】
【请求参数】
【响应结构】
【错误码】
【鉴权与权限】
【幂等与限流】
【兼容性】
【示例】
【验证方式】
【未验证项】
```

## 约束

- 无法确认的字段语义要明确标记为待确认或未验证项。
- 不能只给 happy path，要覆盖错误码和异常场景。
- 如果影响旧客户端，必须说明兼容策略。
- 没有验证路径时，不要把设计写成“已确认无风险”。

## 不适用场景

- 纯 UI 文案或样式调整。
- 无接口边界变化的纯内部重构。

## 使用示例

最小调用示例：

```text
请使用 api-design skill 设计这个接口。

请输出：
1. 接口目标
2. 请求方法与路径
3. 请求参数
4. 响应结构
5. 错误码
6. 兼容性
7. 验证方式
8. 未验证项

要求：
- 不要直接进入服务端实现。
- 不要只给 happy path。
```

更多示例见 `usage-examples.md`。
