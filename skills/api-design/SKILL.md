# 接口设计 Skill

## 适用场景

- 新接口设计
- 旧接口扩展或兼容改造
- 需要明确请求、响应、错误码和版本策略

## 输入

- 业务目标和调用场景
- 调用方类型
- 已有相似接口和兼容约束

## 工作流程

1. 明确业务场景和调用方。
2. 设计接口路径和方法。
3. 设计请求参数。
4. 设计响应结构。
5. 设计错误码。
6. 检查鉴权、权限、幂等、限流。
7. 检查客户端兼容性和版本策略。
8. 输出接口示例。

## 输出格式

```text
【接口目标】
【请求方法与路径】
【请求参数】
【响应结构】
【错误码】
【鉴权与权限】
【幂等与限流】
【兼容性】
【示例】
```

## 约束

- 无法确认的字段语义要明确标记为待确认。
- 不能只给 happy path，要覆盖错误码和异常场景。
- 如果影响旧客户端，必须说明兼容策略。

## 不适用场景

- 纯 UI 文案或样式调整
- 无接口边界变化的纯内部重构
