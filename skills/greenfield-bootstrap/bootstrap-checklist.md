# Greenfield Bootstrap 检查清单

- [ ] 当前要启动的端有哪些：Android / iOS / Backend
- [ ] 是否需要多端并行推进，还是可以按顺序启动
- [ ] 第一阶段是否必须依赖后端接口、鉴权或返回结构
- [ ] 是否已有原型、接口契约、数据结构约束或交互流程图
- [ ] 第一阶段目标是 demo 验证，还是可持续迭代的正式起盘
- [ ] 是否包含登录、支付、推送、上传、埋点、本地存储、离线能力等基础需求
- [ ] 是否已有团队偏好的技术栈、目录规范、命名约定或环境配置方式
- [ ] 是否存在客户端与服务端之间必须先统一的接口边界、DTO 或错误码约束
- [ ] 哪些关键信息仍然未知，必须写入“未验证项”
