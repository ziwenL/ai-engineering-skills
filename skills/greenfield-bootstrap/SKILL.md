# Greenfield Bootstrap Skill

## 适用场景

- 项目还没创建，需要从 0 起盘
- 需要判断这是单端启动还是多端联动启动
- 需要判断 Backend、Android、iOS 的推荐启动顺序
- 需要在真正创建工程前先明确共享命名、接口约束和阶段边界
- 需要给出后续应进入哪些 scaffold Skill 的明确路由

## 输入

- 已确认的产品目标和第一阶段里程碑
- 已知的目标端范围：Android、iOS、Backend
- 已知的产品能力需求，例如登录、支付、上传、推送、本地存储、离线能力
- 已知的团队技术偏好、目录规范、命名约定或交付节奏
- 已有的原型、接口契约、数据结构约束、交互流程图或环境约束
- 项目入口文档和 `skills/common/`
- `bootstrap-checklist.md`
- `usage-examples.md`

## 工作流程

1. 阅读项目入口文档和 `skills/common/`，确认当前目标是创建新项目，而不是继续开发已有工程。
2. 阅读第一阶段目标和约束，确认这是 Backend only、Android only、iOS only，还是多端联动启动。
3. 阅读 `bootstrap-checklist.md`，逐项检查跨端依赖、启动前提和仍未明确的信息。
4. 判断第一阶段是否应默认以后端优先启动，还是允许客户端先起 demo 壳子。
5. 输出多端职责边界、共享命名约定、接口边界、DTO 或错误码一致性要求，以及环境配置约束。
6. 明确第一阶段各端只应做到什么，不应提前展开什么。
7. 给出推荐默认方案、备选方案与取舍，并路由到下一步 Skill。
8. 输出验证方式、未验证项和风险点。

## 输出格式

```text
【当前启动判断】
【推荐启动顺序】
【多端职责边界】
【共享命名与接口约束】
【推荐默认方案】
【备选方案与取舍】
【本阶段不应提前做的事】
【下一步建议 Skill】
【验证方式】
【未验证项】
【风险点】
```

## 约束

- 这是 greenfield 总控分流层，不直接承担 Android、iOS、Backend 的完整脚手架生成。
- 不跳过跨端依赖判断就直接给出启动顺序。
- `bootstrap-checklist.md` 必须作为判断依据显式使用。
- 不把缺少证据的产品或技术假设写成确定结论。
- 未确认的信息必须进入“未验证项”。
- 输出目标是稳定起盘和明确路由，不是一次性展开完整实现。

## 不适用场景

- 项目已经存在，当前只是继续开发已有 Android、iOS 或 Backend 工程。
- Android 已有稳定实现，当前任务只是补齐 iOS 复刻。
- 已经进入某一端的具体编码阶段，且工程结构已经明确。
- 需求、里程碑或目标端范围仍未确认，无法进行启动顺序判断。

## 使用示例

最小调用示例：

```text
请使用 greenfield-bootstrap skill，判断当前项目应如何从 0 起盘。

已知信息：
1. 产品目标
2. 第一阶段里程碑
3. 目标端范围：Android / iOS / Backend
4. 是否已有 API 契约或原型

请输出：
1. 当前启动判断
2. 推荐启动顺序
3. 多端职责边界
4. 下一步建议 Skill
5. 验证方式
6. 未验证项

要求：
- 不要直接展开完整实现。
- 缺信息时不要猜测，写进未验证项。
```

更多示例见 `usage-examples.md`。
