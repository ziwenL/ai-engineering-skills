# 使用示例

## 说明

这个 Skill 适合在一次任务完成后回看哪些经验值得沉淀进仓库，哪些不该沉淀，重点是只保留已验证、可复用、长期有效的信息。

## 最小调用示例

```text
请使用 session-retrospective skill，回顾本次会话。

请输出：
1. 是否需要沉淀
2. 建议沉淀内容
3. 推荐位置
4. 沉淀理由
5. 不应沉淀内容
6. 需要人工确认的问题

要求：
- 只沉淀可复用、已验证、长期有效的信息。
- 不要沉淀隐私、密钥、一次性数据或未经验证的猜测。
```

## 推荐调用示例

```text
请使用 session-retrospective skill，回顾本次任务，并判断哪些内容值得沉淀回 AI Engineering Skills 仓库。

请结合以下输入：
1. 本次任务目标
2. 实际改动
3. 验证结果
4. 本次踩坑经验
5. 相关现有 Skill 或项目文档

请输出：
1. 是否需要沉淀
2. 建议沉淀内容
3. 推荐位置
4. 沉淀理由
5. 不应沉淀内容
6. 建议修改文件
7. 需要人工确认的问题

要求：
- 只保留已验证、可复用、长期有效的经验。
- 区分应该进 common、已有 Skill、新 Skill、入口文件还是项目文档。
- 不要把临时日志、敏感信息或一次性决策写进沉淀内容。
```

## 高约束调用示例

```text
请使用 session-retrospective skill，对本次任务做沉淀判断。

请严格按以下结构输出：
【是否需要沉淀】
【建议沉淀内容】
【推荐位置】
【沉淀理由】
【不应沉淀内容】
【建议修改文件】
【需要人工确认的问题】

要求：
- 只沉淀已验证、可复用、长期有效的信息。
- 不要沉淀隐私、密钥、临时日志、一次性数据和未经验证的猜测。
- 推荐位置必须和信息性质匹配。
```

## 适用提醒

- 适合任务完成后使用。
- 如果任务还没结束、经验还不稳定，不建议提前做沉淀。
- 这一步通常是进入 `skill-authoring` 或更新文档的前置输入。
