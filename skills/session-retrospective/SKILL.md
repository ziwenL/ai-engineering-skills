# 会话沉淀 Skill

## 适用场景

- 每次任务结束后的会话复盘
- 需要判断是否沉淀规则、流程、决策和经验

## 输入

- 本次任务背景
- 实际改动、验证结果和踩坑经验
- 相关项目文档和已有 Skills
- `usage-examples.md`

## 工作流程

1. 回顾本次任务目标、改动、验证结果和问题。
2. 判断哪些内容是可复用、已验证、长期有效的。
3. 判断应沉淀到 common、已有 Skill、新 Skill、入口文件还是项目文档。
4. 明确哪些内容不应该沉淀。
5. 输出建议沉淀内容、位置和需要人工确认的问题。

## 输出格式

```text
【是否需要沉淀】
【建议沉淀内容】
【推荐位置】
【沉淀理由】
【不应沉淀内容】
【建议修改文件】
【需要人工确认的问题】
```

## 约束

- 只沉淀可复用、已验证、长期有效的信息。
- 不沉淀隐私、密钥、临时日志、一次性数据和未经验证的猜测。
- 推荐位置要和信息性质匹配。

## 不适用场景

- 任务尚未完成、没有形成稳定经验的时候。

## 使用示例

最小调用示例：

```text
请使用 session-retrospective skill，回顾本次会话。

请输出：
1. 是否需要沉淀
2. 建议沉淀内容
3. 推荐位置
4. 沉淀理由
5. 不应沉淀内容
6. 需要人工确认的问题

要求：
- 只沉淀可复用、已验证、长期有效的信息。
- 不要沉淀隐私、密钥、一次性数据或未经验证的猜测。
```

更多示例见 `usage-examples.md`。
