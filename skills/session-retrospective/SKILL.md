# 会话沉淀 Skill

## 适用场景

- 每次任务结束后的会话复盘
- 需要判断是否沉淀规则、流程、决策和经验

## 输入

- 本次任务背景
- 实际改动、验证结果和踩坑经验
- 相关项目文档和已有 Skills

## 工作流程

1. 回顾本次任务目标、改动、验证结果和问题。
2. 判断哪些内容是可复用、已验证、长期有效的。
3. 判断应该沉淀到 common、已有 Skill、新 Skill、入口文件还是项目文档。
4. 明确哪些内容不该沉淀。
5. 输出建议沉淀内容、位置和需要人工确认的问题。

## 判断维度

需要判断是否沉淀到：

1. `.ai/ai-engineering-skills/skills/common/`
2. 某个已有 Skill
3. 新增 Skill
4. `CLAUDE.md`
5. `AGENTS.md`
6. `docs/decisions/`
7. `docs/incident-history/`
8. `docs/ai-sessions/`

## 应沉淀

- 可复用工程规则
- 项目长期约束
- 已验证的架构决策
- 反复出现的问题
- 可复用流程
- 新增检查清单
- 事故根因和长期防线

## 不应沉淀

- 密钥、Token、证书
- 用户隐私
- 临时日志
- 一次性数据
- 未验证猜测
- 与当前项目无关的信息

## 输出格式

```text
【是否需要沉淀】
【建议沉淀内容】
【推荐位置】
【沉淀理由】
【不应沉淀内容】
【建议修改文件】
【需要人工确认的问题】
```

## 约束

- 只沉淀可复用、已验证、长期有效的信息。
- 不沉淀隐私、密钥、临时日志、一次性数据和未经验证猜测。
- 推荐位置要和信息性质匹配。

## 不适用场景

- 任务尚未完成、没有形成稳定经验的时候
