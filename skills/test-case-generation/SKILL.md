# 测试用例生成 Skill

## 适用场景

- 基于需求和技术方案生成测试用例
- 补齐主流程、异常流程和回归覆盖

## 输入

- 需求分析
- 技术方案
- 已知兼容性和风险信息

## 工作流程

1. 阅读需求分析。
2. 阅读技术方案。
3. 按功能路径拆测试场景。
4. 补充异常场景。
5. 补充边界场景。
6. 补充兼容性和回归场景。
7. 标记自动化测试和人工测试。

## 输出格式

```text
【正向用例】
【异常用例】
【边界用例】
【兼容性用例】
【回归用例】
【自动化建议】
【人工验证建议】
```

## 约束

- 不要只覆盖 happy path。
- 自动化建议和人工验证建议要区分清楚。
- 无法确认的行为不要伪装成明确用例。

## 不适用场景

- 没有需求分析或技术方案支撑的空泛测试讨论
