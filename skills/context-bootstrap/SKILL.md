# 上下文初始化 Skill

## 适用场景

- 第一次把 AI Engineering Skills 接入业务项目
- 新会话开始时需要快速恢复项目上下文
- AI 尚不清楚项目结构、技术栈、构建方式和开发约束
- 需要生成或更新 `CLAUDE.md` / `AGENTS.md`

## 输入

- `CLAUDE.md` 或 `AGENTS.md`
- `skills/common/`
- README、目录结构、构建脚本
- 核心模块和测试 / CI 配置
- `usage-examples.md`

## 工作流程

1. 读取项目入口文件：`CLAUDE.md` 或 `AGENTS.md`。
2. 阅读 `skills/common/`，了解仓库通用协作规范。
3. 阅读 README、目录结构、构建脚本和测试 / CI 配置。
4. 找出核心模块、关键入口和主要依赖关系。
5. 判断项目技术栈、模块边界、验证方式和开发约束。
6. 输出缺失上下文和建议补充的文档。
7. 不修改代码，除非用户明确要求。

## 输出格式

```text
【项目技术栈】
【主要模块结构】
【构建与测试方式】
【适用 Skill】
【开发约束】
【缺失上下文】
【建议补充文档】
【下一步建议】
```

## 约束

- 不要修改代码，除非用户明确要求。
- 不要猜测无法从代码和文档确认的信息。
- 缺失上下文时要明确列出，而不是自行补全。

## 不适用场景

- 已经完成上下文理解，现在需要直接进入具体实现的阶段。

## 使用示例

最小调用示例：

```text
请使用 context-bootstrap skill，为当前项目建立 AI Coding 上下文。

请先阅读：
1. CLAUDE.md 或 AGENTS.md
2. skills/common/
3. README
4. 主要目录结构
5. 构建脚本
6. 核心架构代码
7. 测试 / CI 配置

然后输出：
1. 项目技术栈判断
2. 项目主要模块结构
3. 适合使用哪些 Skill
4. 当前项目开发约束
5. 还缺少哪些上下文文档

要求：
- 不要修改代码。
- 不要猜测无法从代码确认的信息。
- 如果上下文不足，请明确列出需要补充的文件或信息。
```

更多示例见 `usage-examples.md`。
