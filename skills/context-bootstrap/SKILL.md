# 上下文初始化 Skill

## 适用场景

- 第一次把 AI Engineering Skills 接入业务项目
- 新会话开始时需要理解旧内容
- AI 不清楚项目结构、技术栈、构建方式
- 需要生成或更新 `CLAUDE.md` / `AGENTS.md`

## 输入

- `CLAUDE.md` 或 `AGENTS.md`
- `skills/common/`
- README、目录结构、构建脚本
- 核心模块和测试/CI 配置

## 工作流程

1. 读取入口文件：`CLAUDE.md` 或 `AGENTS.md`。
2. 读取 `skills/common/`。
3. 阅读 README、目录结构、构建脚本。
4. 找出核心模块和相似实现。
5. 判断项目技术栈、架构、测试方式。
6. 输出缺失上下文和建议补充文档。
7. 不修改代码，除非用户明确要求。

## 输出格式

```text
【项目技术栈】
【主要模块结构】
【构建与测试方式】
【适用 Skill】
【项目开发约束】
【缺失上下文】
【建议补充文档】
【下一步建议】
```

## 约束

- 不要修改代码，除非用户明确要求。
- 不要猜测无法从代码和文档确认的信息。
- 缺失上下文要明确列出来，而不是省略不说。

## 不适用场景

- 已经完成上下文理解、现在需要直接做具体实现的阶段
