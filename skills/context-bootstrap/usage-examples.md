# 使用示例

## 说明

这个 Skill 适合在新项目接入、老项目新会话、或者 AI 对项目上下文还不完整时使用。示例重点是让 AI 先建立上下文，而不是直接写代码。

## 最小调用示例

```text
请使用 context-bootstrap skill，为当前项目建立 AI Coding 上下文。

请先阅读：
1. CLAUDE.md 或 AGENTS.md
2. skills/common/
3. README
4. 主要目录结构
5. 构建脚本
6. 核心架构代码
7. 测试 / CI 配置

然后输出：
1. 项目技术栈判断
2. 项目主要模块结构
3. 适合使用哪些 Skill
4. 当前项目开发约束
5. 还缺少哪些上下文文档

要求：
- 不要修改代码。
- 不要猜测无法从代码确认的信息。
- 如果上下文不足，请明确列出需要补充的文件或信息。
```

## 推荐调用示例

```text
请使用 context-bootstrap skill，为当前项目建立后续 AI 协作所需的基础上下文。

请按下面顺序进行：
1. 阅读 CLAUDE.md 或 AGENTS.md
2. 阅读 skills/common/
3. 阅读 README
4. 查看主要目录结构
5. 查看构建脚本、测试脚本、CI 配置
6. 找出核心模块与关键入口代码

请输出：
1. 项目技术栈与运行方式
2. 项目目录和主要模块职责
3. 当前项目的编码 / 架构 / 提交流程约束
4. 现在最适合使用的 Skill 组合
5. 当前上下文还缺少哪些文档或文件
6. 如果我下一步要开始开发，建议先补哪些信息

要求：
- 先理解，不要直接进入实现。
- 所有结论都尽量基于代码或文档证据。
- 无法确认的信息请单独列为“待补充上下文”。
```

## 高约束调用示例

```text
请使用 context-bootstrap skill，为当前项目建立 AI Coding 上下文。

必须先阅读：
1. CLAUDE.md 或 AGENTS.md
2. skills/common/
3. README
4. 主要目录结构
5. 构建脚本
6. 测试 / CI 配置
7. 至少 2 个核心模块入口文件

请严格按以下结构输出：
【项目技术栈】
【主要模块结构】
【构建与测试方式】
【适用 Skill】
【开发约束】
【缺失上下文】
【建议补充文档】
【下一步建议】

要求：
- 不要修改任何代码或配置。
- 不要根据经验补全项目事实。
- 每条结论尽量对应到实际文件或目录。
- 如果信息不足，直接写进【缺失上下文】。
```

## 适用提醒

- 当你想先“让 AI 看懂项目”时，用这个 Skill。
- 当你已经完成上下文建立，准备进入具体需求实现时，应切到 `technical-design` 或具体开发 Skill。
- 这个 Skill 不负责直接产出实现代码。
