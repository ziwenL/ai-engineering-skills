# 监控分析 Skill

## 适用场景

- 线上指标异常分析
- Crash 增长分析
- 错误率、耗时、流量异常排查

## 输入

- 告警信息
- 指标或图表
- 时间范围
- 最近发版和配置变更

## 工作流程

1. 明确异常指标。
2. 确认异常开始时间。
3. 对比最近发版、配置变更、流量变化。
4. 按版本、平台、地区、渠道、机型拆分。
5. 建立假设并验证。
6. 输出结论和下一步动作。

## 输出格式

```text
【异常指标】
【开始时间】
【影响范围】
【拆分维度】
【已确认事实】
【根因假设】
【下一步排查建议】
```

## 约束

- 不允许只看单一指标下结论。
- 必须区分相关性和因果性。

## 不适用场景

- 没有任何监控数据或时间范围的纯猜测
- 已经进入具体代码修复阶段的问题
