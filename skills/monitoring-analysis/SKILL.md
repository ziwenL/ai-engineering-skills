# 监控分析 Skill

## 适用场景

- 线上指标异常分析
- Crash 增长分析
- 错误率、耗时、流量异常排查

## 输入

- 告警信息
- 指标或图表
- 时间范围
- 最近发布和配置变更
- `usage-examples.md`

## 工作流程

1. 明确异常指标。
2. 确认异常开始时间。
3. 对比最近发布、配置变更和流量变化。
4. 按版本、平台、地区、渠道、机型等维度拆分。
5. 建立假设并验证。
6. 输出结论和下一步动作。

## 输出格式

```text
【异常指标】
【开始时间】
【影响范围】
【拆分维度】
【已确认事实】
【根因假设】
【下一步排查建议】
```

## 约束

- 不允许只看单一指标下结论。
- 必须区分相关性和因果性。

## 不适用场景

- 没有任何监控数据或时间范围的纯猜测。
- 已经进入具体代码修复阶段的问题。

## 使用示例

最小调用示例：

```text
请使用 monitoring-analysis skill 分析这个监控异常。

已知信息：
1. 告警内容
2. 指标或图表
3. 时间范围
4. 最近发布和配置变更

请输出：
1. 异常指标
2. 开始时间
3. 影响范围
4. 拆分维度
5. 已确认事实
6. 根因假设
7. 下一步排查建议

要求：
- 不要只看单一指标下结论。
- 必须区分相关性和因果性。
```

更多示例见 `usage-examples.md`。
