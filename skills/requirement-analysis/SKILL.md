# 需求分析 Skill

## 适用场景

- 新需求分析
- 复杂改动前的业务拆解
- 需要先澄清主流程、异常流程和边界条件

## 输入

- 原始需求描述
- 业务背景
- 已知约束和依赖方

## 工作流程

1. 提取业务目标。
2. 拆解用户故事。
3. 梳理主流程。
4. 梳理异常流程。
5. 梳理边界条件。
6. 标记依赖模块和依赖方。
7. 输出待确认问题。
8. 识别初步风险。

## 输出格式

```text
【业务目标】
【用户故事】
【主流程】
【异常流程】
【边界条件】
【依赖模块】
【待确认问题】
【初步风险】
```

## 约束

- 不要直接写代码。
- 不要直接进入技术方案。
- 无法确认的信息必须标记为“待确认”。

## 不适用场景

- 已经进入具体实现和调试阶段的任务
- 只有代码细节调整、没有业务变化的纯技术修补
