# 线上 Bug 定位 Skill

## 适用场景

- 线上用户反馈定位
- Crash 或错误日志排查
- 发版后异常行为回归定位

## 输入

- 用户反馈和复现线索
- Crash、日志、发版记录、相关 commit
- 配置变更信息
- `log-analysis.md`
- `incident-template.md`

## 工作流程

1. 明确问题现象。
2. 判断影响范围。
3. 阅读 `log-analysis.md`，按时间范围、版本对比和正常/异常用户对比收集证据。
4. 使用 `incident-template.md` 整理问题现象、影响范围、已确认事实和后续动作。
5. 收集日志、Crash、发版、commit、配置变更。
6. 建立根因假设。
7. 验证假设。
8. 给出止血方案、修复方案和回滚建议。
9. 标记证据不足的部分。

## 输出格式

```text
【问题现象】
【影响范围】
【时间范围】
【已确认事实】
【根因假设】
【证据】
【止血方案】
【修复方案】
【回滚建议】
【待补充信息】
```

## 约束

- 证据不足时不允许编造根因。
- 生产数据只读。
- 高风险修复必须人工审批。

## 不适用场景

- 还没有任何现象、日志或影响信息的纯猜测问题
- 需要直接改生产数据的问题
