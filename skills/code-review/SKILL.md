# 代码审查 Skill

## 适用场景

- 审查本次 diff
- 检查需求符合性和架构一致性
- 识别安全、性能、测试和可维护性问题

## 输入

- 任务背景
- 需求分析和技术方案
- 代码 diff
- 相关测试结果

## 工作流程

1. 阅读任务背景。
2. 阅读 diff。
3. 对照需求和技术方案。
4. 检查是否修改无关文件。
5. 检查架构一致性。
6. 检查错误处理、边界条件、测试覆盖。
7. 输出 Review 结论。

## 输出格式

```text
【结论】
【主要问题】
【次要问题】
【测试缺口】
【风险点】
【建议修改】
```

## 约束

- 不能只复述改动，要指出真正的风险和缺口。
- 如果证据不足，要明确写成待确认，不要硬下结论。
- 高风险问题要优先于样式问题。

## 不适用场景

- 没有 diff 的纯需求讨论
- 需要直接实施改动而不是先审查的场景
