# 使用示例

## 说明

这个 Skill 适合在改动后做安全审查，重点看密钥泄露、权限绕过、注入、越权、隐私日志和敏感信息风险。

## 最小调用示例

```text
请使用 security-review skill 审查本次改动。

请重点关注：
1. 密钥 / Token / 证书泄露
2. 越权访问
3. 注入风险
4. 权限或鉴权绕过
5. 日志中的隐私数据

请输出：
1. 审查结论
2. 主要风险
3. 风险等级
4. 证据
5. 修复建议
6. 是否阻塞合并 / 发布
```

## 推荐调用示例

```text
请使用 security-review skill 审查当前改动。

请基于以下输入：
1. 代码 diff 或关键文件
2. 需求背景和接口边界
3. 已知风险点
4. 相关配置、日志或测试结果

请输出：
1. 审查结论
2. 主要风险
3. 风险等级
4. 受影响文件
5. 证据
6. 修复建议
7. 待确认问题
8. 是否阻塞合并 / 发布

要求：
- 不要把“没看到问题”写成“绝对安全”。
- 高风险问题优先于低影响问题。
- 没有证据时，不要伪装成确定漏洞。
```

## 高约束调用示例

```text
请使用 security-review skill，对当前改动进行严格安全审查。

请严格按以下结构输出：
【审查结论】
【主要风险】
【风险等级】
【受影响文件】
【证据】
【修复建议】
【待确认问题】
【是否阻塞合并 / 发布】

要求：
- 没有证据时不要伪装成确定漏洞。
- 高风险问题优先于样式和低影响问题。
- 不要把“没看到问题”当成“绝对安全”。
```

## 适用提醒

- 适合高风险改动、发布前、或涉及鉴权和敏感数据时使用。
- 如果是一般代码质量审查，优先用 `code-review`。
- 如果需要真正渗透测试或攻防验证，这个 Skill 不足以替代专业手段。
