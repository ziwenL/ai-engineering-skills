# Backend Service Scaffold 检查清单

- [ ] 当前第一阶段是否只需要单服务骨架，而不是提前拆分微服务
- [ ] 是否需要数据库、缓存、消息队列、对象存储或第三方服务占位
- [ ] 是否需要鉴权、健康检查、统一错误处理、结构化日志和基础监控占位
- [ ] 是否已有接口契约、错误码约定、返回结构规范或客户端依赖的字段约束
- [ ] 是否存在环境配置约束，例如本地、测试、预发、生产的差异
- [ ] 第一阶段是否只需要最小可运行 API 壳子，而不是完整业务链路
- [ ] 团队是否已有必须沿用的语言、框架、目录规范、测试方案或部署约束
- [ ] 哪些信息仍不明确，必须写入未验证项
