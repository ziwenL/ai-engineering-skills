# iOS App Scaffold 检查清单

- [ ] 当前是否只需要单工程起步，而不是一开始就引入 `workspace`、多 `target` 或额外 `package`
- [ ] 第一阶段是否只需要最小可运行壳子，而不是完整业务页面与完整流程
- [ ] 是否需要预留网络层、登录态、本地持久化、埋点或导航占位
- [ ] 是否存在未来与 Android 对齐的数据结构、交互行为或错误处理约束
- [ ] 是否已有团队偏好的工程组织方式、命名规则、状态管理方式或依赖管理约束
- [ ] 第一阶段是否必须支持真机演示、测试包验证或仅需 Simulator 验证
- [ ] 哪些信息仍不明确，必须写入未验证项
