# 前端开发 Skill

## 适用场景

- Web、H5、管理后台功能开发
- 页面或组件 Bug 修复
- 状态管理、接口接入和交互状态调整

## 输入

- 已确认的需求分析
- 已确认的技术方案
- 相似页面、组件和状态管理方式
- 项目入口文件和 `skills/common/`
- `usage-examples.md`

## 工作流程

1. 阅读需求分析和技术方案。
2. 阅读项目入口文件和 `skills/common/`，确认当前项目约束。
3. 查找相似页面、组件、状态管理方式。
4. 先列出计划修改文件和原因。
5. 实现 UI、状态、接口和错误处理。
6. 覆盖 loading、empty、error、success，以及必要时的 retry、permission denied、timeout。
7. 检查埋点、兼容性、可访问性和回归影响。
8. 验证构建、Lint、测试。
9. 输出风险、验证方式和未验证项。

## 输出格式

```text
【修改文件列表】
【每个文件的修改原因】
【核心实现逻辑】
【状态与异常处理】
【接口接入变化】
【验证命令与结果】
【未验证项】
【风险点】
【人工验证建议】
```

## 约束

- 不做无关重构。
- 不引入新依赖，除非先得到确认。
- 不能只实现 happy path，必须覆盖主要异常状态。
- 没有运行验证命令时，不能声称“已验证通过”。

## 不适用场景

- 纯服务端、纯 Android 或纯 iOS 任务。
- 需求和方案尚未确认的任务。

## 使用示例

最小调用示例：

```text
请使用 frontend-development skill 实现该需求。

请先阅读：
1. 已确认的需求分析
2. 已确认的技术方案
3. 项目入口文档
4. skills/common/
5. 相似页面、组件和状态管理实现

要求：
- 先列出计划修改的文件和原因。
- 不要只实现 happy path。
- 实现后运行构建、Lint 或测试命令。

最后输出：
1. 修改摘要
2. 验证结果
3. 未验证项
4. 风险点
```

更多示例见 `usage-examples.md`。
