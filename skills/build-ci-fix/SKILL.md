# 构建与 CI 修复 Skill

## 适用场景

- 编译失败
- 测试失败
- Lint 或格式检查失败
- CI 环境和本地环境不一致

## 输入

- 失败日志
- 最近相关改动
- 本地可运行的对应命令

## 工作流程

1. 阅读失败日志。
2. 定位失败阶段：编译 / 测试 / Lint / 打包 / 发布。
3. 提取最小错误信息。
4. 查找相关文件和最近改动。
5. 最小化修复。
6. 重新运行失败命令。
7. 不允许为了通过 CI 删除测试或绕过检查，除非用户明确确认。

## 输出格式

```text
【失败阶段】
【关键错误】
【根因假设】
【最小修复方案】
【验证命令】
【验证结果】
【风险点】
```

## 约束

- 不允许为了通过 CI 删除测试。
- 不允许随意降低 Lint 或检查规则。
- 没有复跑验证前，不要声称“CI 已修复”。

## 不适用场景

- 没有日志和失败命令的模糊问题
- 与构建无关的纯业务需求开发
