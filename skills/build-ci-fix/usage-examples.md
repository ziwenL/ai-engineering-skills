# 使用示例

## 说明

这个 Skill 适合在编译、测试、Lint 或 CI 环境失败时，先定位失败阶段、复现问题，再给出最小修复方案和验证结果。

## 最小调用示例

```text
请使用 build-ci-fix skill 分析这段 CI 失败日志，并给出最小修复方案。

请输出：
1. 失败阶段
2. 关键错误
3. 根因假设
4. 最小修复方案
5. 验证命令
6. 验证结果
7. 未验证项

要求：
- 不要为了通过 CI 删除测试。
- 不要随意降低 Lint 或检查规则。
```

## 推荐调用示例

```text
请使用 build-ci-fix skill，定位并修复当前构建 / CI 失败。

请基于以下输入：
1. 失败日志
2. 最近相关改动
3. 本地可运行的对应命令

请输出：
1. 失败阶段
2. 关键错误
3. 根因假设
4. 最小修复方案
5. 验证命令
6. 验证结果
7. 未验证项
8. 风险点

要求：
- 先定位失败阶段，再判断是代码问题、依赖问题还是环境差异。
- 必须尽量本地复现并复跑验证。
- 不允许为了通过 CI 删除测试或随意降规则。
```

## 高约束调用示例

```text
请使用 build-ci-fix skill，严格分析并修复当前 CI 失败。

请严格按以下结构输出：
【失败阶段】
【关键错误】
【根因假设】
【最小修复方案】
【验证命令】
【验证结果】
【未验证项】
【风险点】

要求：
- 不要为了通过 CI 删除测试。
- 不要随意降低 Lint 或检查规则。
- 没有复跑验证前，不要声称“CI 已修复”。
```

## 适用提醒

- 适合构建链、测试链、Lint 链失败时使用。
- 如果问题与构建无关而是纯业务设计问题，不适合用它。
- 它的目标是最小修复，不是顺手做大重构。
