# 通用工程原则

## 基本原则

1. 先理解需求，再设计方案，再修改代码。
2. 优先最小可行改动，不做无关重构。
3. 优先复用现有架构和已有模块。
4. 不引入新依赖，除非用户明确确认。
5. 不修改与任务无关的文件。
6. 所有修改必须可验证。
7. 无法验证时，必须明确说明原因和人工验证方式。
8. 证据不足时，不要编造结论。

## 页面状态原则

涉及页面、接口、流程状态时，必须考虑：

- loading
- success
- empty
- error
- retry
- permission denied
- network unavailable
- timeout
- partial success

## 高风险改动

以下改动必须先出方案并等待人工确认：

- 登录鉴权
- 支付
- 权限系统
- 账号系统
- 数据删除
- 数据库迁移
- 生产配置
- 发版流程
- 安全策略
