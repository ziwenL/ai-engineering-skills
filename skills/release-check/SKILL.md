# 发布检查 Skill

## 适用场景

- 发布前风险检查
- 灰度和回滚准备
- 需要确认测试、监控和高风险模块状态

## 输入

- 发布内容和变更范围
- 测试结果
- 风险模块清单
- 灰度和回滚信息
- `usage-examples.md`

## 工作流程

1. 梳理发布内容和影响范围。
2. 检查测试结果、风险模块和依赖变更。
3. 确认灰度方案、回滚方案和发布后监控指标。
4. 给出发布建议和阻塞项。

## 输出格式

```text
【发布内容】
【测试结果】
【风险模块】
【灰度方案】
【回滚方案】
【监控指标】
【发布建议】
```

## 约束

- 高风险模块不能只写“已测试”，要写清验证依据。
- 如果没有灰度或回滚方案，要明确标记风险。
- 不要为了赶发布隐藏阻塞问题。

## 不适用场景

- 与发布无关的日常开发任务。
- 没有发布范围和测试结果的空白评估。

## 使用示例

最小调用示例：

```text
请使用 release-check skill，检查这次发布是否准备完成。

请输出：
1. 发布内容
2. 测试结果
3. 风险模块
4. 灰度方案
5. 回滚方案
6. 监控指标
7. 发布建议

要求：
- 不要为了赶发布隐藏阻塞问题。
- 没有灰度或回滚方案时要明确标风险。
```

更多示例见 `usage-examples.md`。
