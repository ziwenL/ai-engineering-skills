# 使用示例

## 说明

这个 Skill 适合在发布前做最后一轮检查，确认变更范围、测试结果、灰度方案、回滚方案和监控指标是否齐备。

## 最小调用示例

```text
请使用 release-check skill，检查这次发布是否准备完成。

请输出：
1. 发布内容
2. 测试结果
3. 风险模块
4. 灰度方案
5. 回滚方案
6. 监控指标
7. 发布建议

要求：
- 不要为了赶发布隐藏阻塞问题。
- 没有灰度或回滚方案时要明确标风险。
```

## 推荐调用示例

```text
请使用 release-check skill，对当前发布做最终检查。

请基于以下输入：
1. 发布内容和变更范围
2. 测试结果
3. 风险模块清单
4. 灰度信息
5. 回滚信息

请输出：
1. 发布内容
2. 测试结果
3. 风险模块
4. 灰度方案
5. 回滚方案
6. 监控指标
7. 发布建议

要求：
- 高风险模块不能只写“已测试”，要写清验证依据。
- 如果没有灰度或回滚方案，要明确标记风险。
- 不要为了赶发布隐藏阻塞问题。
```

## 高约束调用示例

```text
请使用 release-check skill，严格检查当前发布准备情况。

请严格按以下结构输出：
【发布内容】
【测试结果】
【风险模块】
【灰度方案】
【回滚方案】
【监控指标】
【发布建议】

要求：
- 高风险模块不能只写“已测试”，必须说明验证依据。
- 没有灰度或回滚方案时要明确标记风险。
- 不要为了赶发布隐藏阻塞问题。
```

## 适用提醒

- 适合发布前做最终检查。
- 如果还在日常开发阶段、没有明确发布范围，不适合用它。
- 若检查中暴露高风险问题，通常需要转回 `code-review`、`security-review` 或 `performance-review` 进一步确认。
